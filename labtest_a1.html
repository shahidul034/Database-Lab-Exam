  • The Departments table has 2 columns: department_id and name.
    • The department_id column is an integer column that is set as the primary key of the table.
  • The Employees table has 6 columns: employee_id, first_name, last_name, email, hire_date, and department_id.
    • The employee_id column is an integer column and is set as the primary key of the table.
    • The department_id column is an integer column that is a foreign key referencing the department_id column in the Departments table.
  • The Salaries table has 4 columns: employee_id, salary, from_date, and to_date.
    • The employee_id column is an integer column that is a foreign key referencing the employee_id column in the Employees table.
    • The from_date column is a DATE column that stores the start date for the salary record. This column, together with employee_id, forms the primary key of the table.
